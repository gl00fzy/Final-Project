# Flow การทำงานระบบตรวจข้อสอบแบบปรนัย (Multiple Choice Grading System)

ระบบ Web Application สำหรับตรวจข้อสอบปรนัย (5 ตัวเลือก) ด้วยการสแกนผ่านกล้องสมาร์ทโฟน พัฒนาด้วยภาษา PHP และประมวลผลผ่าน Web Browser พร้อมฟังก์ชันแชร์เฉลย และบันทึกคะแนนเข้าฐานข้อมูลแบบ Real-time

---

## 1. Flow การทำงานหลักของระบบ (System Workflow)

### 1.1 การสร้างและจัดการเฉลยข้อสอบ (Key Management & Sharing)
1. **อาจารย์ผู้สอนเข้าสู่ระบบ (Login):** ผ่าน Web Application บนมือถือหรือคอมพิวเตอร์
2. **สร้างชุดข้อสอบ (Create Exam):** กำหนดชื่อวิชา, รหัสวิชา, และจำนวนข้อสอบ (รองรับ 50, 100, หรือ 150 ข้อ)
3. **สร้างเฉลย (Create Answer Key):** - อาจารย์ทำการเลือกคำตอบที่ถูกต้อง (A, B, C, D, E) สำหรับแต่ละข้อ
   - ระบบบันทึก Answer Key ลงในฐานข้อมูล
4. **การแชร์เฉลย (Share Key):**
   - อาจารย์สามารถกดปุ่ม "Share" ระบบจะสร้าง Shareable Link หรือรหัส PIN 
   - ส่งให้อาจารย์ผู้สอนร่วมท่านอื่น เพื่อให้สามารถดึง Key นี้ไปใช้ตรวจในบัญชีของตนเองได้

### 1.2 การสแกนกระดาษคำตอบ (Scanning Flow)
1. **เปิดโหมดตรวจข้อสอบ:** อาจารย์เปิดหน้า "สแกนกระดาษคำตอบ" บนมือถือ ระบบจะขออนุญาตเข้าถึงกล้อง (WebRTC / HTML5 Camera API)
2. **การถ่าย/สแกน (Scan):**
   - นำกล้องมือถือไปจ่อที่กระดาษคำตอบ (1 แผ่น A4)
   - ระบบ (ใช้ JavaScript เช่น OpenCV.js ในฝั่ง Client หรือส่งภาพไปให้ PHP/Python ฝั่ง Server) จะทำการค้นหาจุดอ้างอิง (Fiducial Markers) ทั้ง 4 มุมของกระดาษ
3. **การประมวลผลภาพ (Image Processing):**
   - อ่านการฝนรหัสนิสิต (11 หลัก)
   - อ่านการฝนคำตอบ และเปรียบเทียบกับ Answer Key
4. **การตอบกลับ (Feedback & Alert):**
   - เมื่อระบบอ่านครบและประมวลผลสำเร็จ จะส่งเสียงเตือน **"Peep"** (ผ่าน HTML5 Audio) ทันที
   - หน้าจอจะแสดงหน้าต่าง Pop-up หรือ Overlay สรุปข้อมูล: `รหัสนิสิต`, `คะแนนที่ได้`

### 1.3 การบันทึกข้อมูล (Data Storage Flow)
1. **ยืนยันการบันทึก:** หลังตรวจเสร็จ ระบบจะส่งข้อมูลผ่าน AJAX (JavaScript) ไปยัง Backend (PHP)
2. **บันทึกลงฐานข้อมูล (Database):**
   - PHP รับค่า รหัสนิสิต, คะแนนที่ได้, รหัสข้อสอบ และบันทึกลง MySQL/MariaDB
3. **แสดงผลรอบถัดไป:** หน้าจอเคลียร์ค่าเดิม และพร้อมสแกนกระดาษคำตอบแผ่นต่อไปทันที (Continuous Scanning)

## 2. การออกแบบกระดาษคำตอบ (Answer Sheet Design)

ข้อจำกัด: ต้องอยู่ในขนาด **1 หน้า A4** เท่านั้น ระบบรองรับจำนวนข้อที่ยืดหยุ่น (50, 100, 150 ข้อ)

### โครงสร้างบนหน้ากระดาษ (Layout):
* **จุดอ้างอิง (Markers):** สี่เหลี่ยมสีดำทึบที่มุมทั้ง 4 ด้านของกระดาษ สำหรับให้ระบบ Align ภาพ
* **ส่วนที่ 1: ข้อมูลส่วนตัว (Text Fields)**
    * ช่องว่างสำหรับเขียน: ชื่อ - นามสกุล
    * ช่องว่างสำหรับเขียน: เบอร์โทรศัพท์
* **ส่วนที่ 2: รหัสนิสิต (Student ID)**
    * ช่องว่างสำหรับเขียนรหัสนิสิต 11 หลัก เป็นตัวเลข
    * ตารางฝนรหัสนิสิต: 11 คอลัมน์ x 10 แถว (เลข 0-9)
* **ส่วนที่ 3: พื้นที่ฝนคำตอบ (Answer Grid)**
    * ข้อสอบแบบ 5 ตัวเลือก (เช่น ก, ข, ค, ง, จ หรือ 1, 2, 3, 4, 5)
    * **กรณี 50 ข้อ:** แบ่งเป็น 2 คอลัมน์ คอลัมน์ละ 25 ข้อ (มีพื้นที่เหลือเฟือ ช่องกว้าง)
    * **กรณี 100 ข้อ:** แบ่งเป็น 4 คอลัมน์ คอลัมน์ละ 25 ข้อ (ขนาดช่องมาตรฐาน)
    * **กรณี 150 ข้อ:** แบ่งเป็น 6 คอลัมน์ คอลัมน์ละ 25 ข้อ (ขนาดช่องเล็กกะทัดรัด แต่ยังสามารถฝนและสแกนได้ใน 1 หน้า A4)

## 3. โครงสร้างเทคโนโลยี (Tech Stack)

* **Frontend (ฝั่งผู้ใช้งานบนมือถือ):**
    * HTML5 / CSS3 (Responsive Design รองรับจอสมาร์ทโฟน)
    * JavaScript (AJAX) สำหรับจัดการ Flow แบบไม่ต้องรีเฟรชหน้า
    * `getUserMedia` API สำหรับดึงภาพจากกล้อง
    * `OpenCV.js` (แนะนำ) สำหรับจับจุดมาร์คเกอร์และอ่านค่าเบื้องต้นแบบ Real-time เพื่อความรวดเร็ว
    * `Audio Object` ใน JS สำหรับส่งเสียง Peep `new Audio('peep.mp3').play();`
* **Backend (ฝั่งเซิร์ฟเวอร์):**
    * **PHP:** จัดการ Logic, ตรวจสอบสิทธิ์, คำนวณคะแนน (กรณีส่งภาพมาประมวลผลฝั่งเซิร์ฟเวอร์) และจัดการ API
* **Database:**
    * MySQL หรือ PostgreSQL

## 4. โครงสร้างฐานข้อมูลเบื้องต้น (Database Schema)

**1. Table: `users`** (เก็บข้อมูลอาจารย์)
- `user_id` (PK)
- `username` / `password` / `name`

**2. Table: `exams`** (เก็บชุดข้อสอบและ Key)
- `exam_id` (PK)
- `owner_id` (FK -> users.user_id)
- `exam_title`
- `question_count` (50, 100, 150)
- `answer_key` (JSON format เช่น `{"1":"A", "2":"C", ...}`)

**3. Table: `exam_shares`** (จัดการการแชร์ให้สอนร่วม)
- `share_id` (PK)
- `exam_id` (FK -> exams.exam_id)
- `shared_to_user_id` (FK -> users.user_id)

**4. Table: `student_scores`** (เก็บผลคะแนน)
- `score_id` (PK)
- `exam_id` (FK -> exams.exam_id)
- `student_id` (VARCHAR 11)
- `score` (INT)
- `scanned_by` (FK -> users.user_id)
- `scanned_at` (TIMESTAMP)
