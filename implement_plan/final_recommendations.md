# Final Recommendations for Multiple Choice Grading System (Production-Ready)

เอกสารฉบับนี้รวบรวมข้อเสนอแนะเชิงเทคนิคเพื่อยกระดับโปรเจกต์จบ (Thesis/Final Project) ให้มีความปลอดภัย เสถียร และพร้อมสำหรับการนำเสนอต่อมหาวิทยาลัยเพื่อใช้งานจริง

---

## 1. การยกระดับความปลอดภัย (Security Hardening)
เนื่องจากระบบเกี่ยวข้องกับคะแนนและข้อมูลนิสิต ความปลอดภัยจึงเป็นสิ่งสำคัญอันดับแรก

* **Database Security:** * ใช้ **PDO (PHP Data Objects)** พร้อมกับ **Prepared Statements** ในทุก SQL queries เพื่อป้องกัน SQL Injection 100%
    * **Password Hashing:** เข้ารหัสรหัสผ่านอาจารย์ด้วย `password_hash(pwd, PASSWORD_DEFAULT)` และตรวจสอบด้วย `password_verify()` ห้ามเก็บเป็น plain text เด็ดขาด
* **Infrastructure:**
    * **HTTPS Only:** เมื่อนำขึ้นระบบจริง ต้องติดตั้ง SSL Certificate เพื่อเข้ารหัสข้อมูลระหว่างมือถืออาจารย์และเซิร์ฟเวอร์
* **Access Control:**
    * ตรวจสอบ Session ทุกหน้าเพจ เพื่อป้องกันการเข้าถึง Dashboard โดยไม่ได้รับอนุญาต
    * ระบบ Share Key ต้องตรวจสอบสิทธิ์ว่าผู้รับมีสิทธิ์เข้าถึงข้อสอบชุดนั้นจริงหรือไม่

## 2. ความเสถียรและการรองรับผู้ใช้งาน (Scalability & Stability)
เพื่อให้ระบบทำงานได้ลื่นไหลแม้มีการแสกนพร้อมกันจำนวนมาก

* **Database Choice:** * เปลี่ยนจาก SQLite เป็น **MySQL หรือ MariaDB** สำหรับการใช้งานจริง เพื่อรองรับการเขียนข้อมูลพร้อมกัน (Concurrent Write) จากอาจารย์หลายท่าน
* **Performance:**
    * ทำ **Indexing** ในฐานข้อมูลที่คอลัมน์ `student_id` และ `exam_id` ในตารางคะแนน เพื่อความรวดเร็วในการออกรายงาน
* **Memory Management (OpenCV.js):**
    * ต้องทำการสั่ง `mat.delete()` ทุกครั้งที่ประมวลผลเฟรมภาพเสร็จ เพื่อป้องกัน Memory Leak ซึ่งจะทำให้ Browser บนมือถือค้างหรือเด้งออก

## 3. ความโปร่งใสและการตรวจสอบ (Audit & Integrity)
สิ่งที่จะทำให้มหาวิทยาลัยยอมรับระบบ คือความสามารถในการตรวจสอบย้อนหลัง

* **Audit Logs:** สร้างตารางบันทึกประวัติ (Log) เมื่อมีการเพิ่ม แก้ไข หรือลบคะแนน โดยเก็บข้อมูล ID ผู้ทำ, เวลา, และ IP Address
* **Manual Override Handling:** หากมีการแก้ไขคะแนนด้วยมือ (Manual) ระบบต้องบันทึกเหตุผลและทำเครื่องหมายไว้ในระบบเพื่อให้ตรวจสอบได้ภายหลัง

## 4. ฟีเจอร์สำหรับการใช้งานจริง (Production Features)
* **Duplicate Prevention:** ระบบต้องเช็ค Student ID + Exam ID ก่อนบันทึก หากพบข้อมูลซ้ำต้องแจ้งเตือนทันที (ป้องกันการแสกนกระดาษใบเดิมซ้ำ)
* **Exporting Data:** เพิ่มปุ่ม **Export to Excel/CSV** เพื่อให้อาจารย์ดาวน์โหลดคะแนนไปตัดเกรดในระบบของมหาลัยได้ทันที
* **Error Feedback:** * **Success:** เสียง Peep + กรอบสีเขียว
    * **Error:** เสียง Alert (คนละโทน) + กรอบสีแดง พร้อมข้อความบอกสาเหตุ (เช่น "หาจุด Marker ไม่ครบ" หรือ "แสงไม่พอ")

---

## 5. แผนการนำเสนอ (Pitching Plan)
เมื่อนำไปเสนอทางมหาวิทยาลัย ควรเน้นย้ำจุดเด่น:
1.  **Cost-Effective:** ไม่ต้องซื้อเครื่องตรวจกระดาษราคาแพง ใช้แค่มือถืออาจารย์
2.  **Real-time:** รู้ผลทันที นิสิตไม่ต้องรอนาน
3.  **Scalable:** รองรับข้อสอบได้ถึง 150 ข้อในแผ่นเดียว และแชร์เฉลยระหว่างผู้สอนร่วมได้ง่าย
